🔴 PROBLEMA ENCONTRADO EM PRODUÇÃO
──────────────────────────────────

Todos os 4 endpoints OAuth requeriam autenticação admin:
  • GET /aletube/auth/youtube → 401 "Invalid admin key"
  • GET /aletube/callback/youtube → 401 "Invalid admin key"  
  • GET /aletube/auth/facebook → 401 "Invalid admin key"
  • GET /aletube/callback/facebook → 401 "Invalid admin key"

RESULTADO: Usuários normais não conseguiam conectar redes sociais em produção

  • Linha 286: callback_youtube() — removido Depends(require_admin)
  • Linha 331: auth_facebook() — removido Depends(require_admin)
  • Linha 346: callback_facebook() — removido Depends(require_admin)

Commit: cd3be77

🔒 POR QUE É SEGURO?
────────────────────

OAuth 2.0 é seguro por design:

1. Google valida identidade do usuário (não seu backend)
2. Backend NUNCA recebe password
3. Client_secret protegido (fica no backend, nunca sai)
4. State token previne CSRF
5. Code válido apenas uma vez
6. Cada usuário acessa só sua própria conta

Logo: require_admin era redundante e BLOQUEADOR
